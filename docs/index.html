Abstract Interpretation in Practice

A survey of industrial static analysis tools and compiler optimizations based on the theory of sound approximation.

Industrial Static Analysis Tools

Astr√©e

Proprietary
A sound static analyzer that proves the absence of runtime errors in C/C++ applications. Famous for verifying the primary flight control software of the Airbus A340/A380 with zero false alarms.
Target: Embedded C/C++
Use Case: Avionics, Automotive

Frama-C (Eva)

Open Source
A platform for source code analysis of C software. The Eva plugin uses abstract interpretation to verify C code against ACSL specifications, computing precise variation domains.
Target: C
Developer: CEA-List, Inria

Facebook Infer

Open Source
Scalable static analysis for mobile and server code. Uses Infer.AI to build abstract interpretation-based checkers. It enables compositional analysis (bi-abduction) to handle huge codebases.
Target: Java, C++, Obj-C
Use Case: Mobile, Web

Polyspace

Proprietary
One of the earliest commercial tools. Verifies C/C++ and Ada for runtime errors and MISRA compliance. Uses color-coding to visualize safe, unsafe, dead, and unproven code.
Target: C, C++, Ada
Developer: MathWorks

IKOS

Open Source
Inference Kernel for Open Static Analyzers. A high-performance framework from NASA for building static analyzers based on abstract interpretation.
Target: C, C++
Developer: NASA Ames

Compiler Phases & Optimizations

Strictness Analysis

GHC
Used in lazy functional compilers (like GHC) to identify arguments that are always evaluated. Enables call-by-value optimization, removing thunk overhead. Formulated as backward abstract interpretation.
Type: Backward Analysis
Goal: Performance

Value Range Propagation

GCC, LLVM
Determines possible value ranges for variables. Enables removal of redundant checks and dead code elimination. A direct application of Interval Analysis.
Type: Forward Analysis
Goal: Optimization

Polyhedral Optimization

Polly, Graphite
Models loop nests as points in a polyhedron to perform complex transformations (tiling, vectorization). deeply rooted in geometric abstractions similar to abstract interpretation.
Type: Loop Transformation
Goal: Parallelism/Locality
